# 캘린더봄 변경 기록

## v0.2.0 — 2026-07-15

70대 독립 사용 우선 UX·맞춤 일정 엔진 업그레이드 (V6).

- 아코디언 일정 추가: 완료한 선택은 한 줄로 접히고(✓+변경) 지금 필요한 질문 하나만 펼쳐집니다. 주 CTA는 항상 하나.
- 질문 예산: 일반 일정 결정 2개, 병원 2~3개, 약 첫 등록 3~4개, 최근 재사용 1~2개 (docs/QUESTION-BUDGET.md).
- 지난번과 같게·최근 사용(2개)·일정 복사·다음 일정 만들기 — 반복 입력 제거.
- 시간 선택 재설계: 큰 현재 시간 + 빠른 칩 최대 4개 + −10분/＋10분/정각 + OS 시간 선택기. 시·분 버튼 18개 나열 제거.
- 사람(지연 등록·이름만): 기본은 "나", 약속·기념일 등 필요한 일정에서만 질문.
- 반복 일정 엔진: 매일·매주(요일)·매월(월말 당김)·매년(윤일 당김), 수정·삭제 범위 = 오늘만/앞으로/전체.
- 약 먹기: 하루 1·2·3번 시간 슬롯(기본 아침8·점심1·저녁7), 이름 입력 없이 저장, 먹었어요/조금 있다가(5·10·30·60분)/건너뛰었어요, 달력엔 묶음 1개 표시. 의료 판단 없음.
- 생일·기념일: 100일·200일·1주년 + N일/N개월/N주년 직접 계산("4월 10일을 1일째로 계산해요"), 고른 날만 생성, 하루 종일 일정과 알림 시각 분리(7일 전/하루 전/당일 오전).
- 일정 한 건 공유: 미리보기 후 문자/ICS — 다른 일정·전체 달력·메모는 절대 포함 안 됨. 약 공유 시 민감정보 안내.
- 실수 복구: 저장·삭제 후 [되돌리기], 작성 중 초안 자동 보존·복원.
- 데이터 v2 + 안전 마이그레이션: v1 일정 전부 보존(원본 키 유지·fallback), JSON v2/ICS 내보내기·가져오기(v1 파일도 수용).
- 검증: 단위 테스트 31 + 정적 검증 + E2E 60항목(과제 A~E·마이그레이션·320~1280px·아주 크게·200% 확대·키보드) 통과.

## v0.1.0 — 2026-07-15

첫 공개 버전.

- 큰 월간 달력: 날짜를 누르면 그날의 일정 확인과 빠른 추가 시트가 열립니다.
- 키보드 없는 일정 저장: 내용 버튼(병원·약 먹기·가족 모임…) → 오전/오후·시·분 버튼 → 알람 시점 → 저장, 최대 4단계.
- 자연어 확인: 저장하면 "7월 20일 월요일 오후 3시, '병원 예약' 알람을 저장했어요."라고 알려줍니다.
- 알림 탭: 다가오는 일정을 D-day(오늘/내일/D-n)와 함께 시각순으로 표시, 완료·삭제 관리.
- 알람: 페이지가 열려 있는 동안 정각·10분 전·1시간 전·하루 전에 브라우저 알림(권한 허용 시)과 화면 안내로 알려줍니다. 한계는 설정 화면에 정직하게 안내.
- 글자 크기 설정: 보통/크게/아주 크게 — 달력·버튼까지 함께 커집니다.
- 보관·이동: JSON 보관 파일과 ICS 달력 파일 내보내기·가져오기(중복 자동 제외).
- PWA: 오프라인 동작(네트워크 우선 캐시), 홈 화면 설치, iOS 대응.
- 접근성: 터치 목표 48px 이상, 320px 화면 무가로스크롤, 다크 모드에서도 밝은 화면 유지, aria-label·aria-live.
- 데이터는 기기 안에만 저장(localStorage). 서버 전송·계정 없음.
