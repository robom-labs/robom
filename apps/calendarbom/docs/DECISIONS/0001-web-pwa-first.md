# 결정 0001 — 캘린더봄 1차 출시는 웹 PWA 로 한다

- 날짜: 2026-07-15
- 상태: 채택
- 결정자: Fable 5 (총괄팀 실행형 요청, COMPANY-MODE 위임 범위 내)

## 배경

원 요구 문서(`calendarbom_codex_ultimate_prompt.md`)는 안드로이드 네이티브(Codex 대상) 프롬프트였다.
그러나 회장 지시는 "네 판단을 넣어 잘 만들고, 기존 다른 앱들과 같은 형태의 실행 주소로 달라"였다.

## 결정

1차 출시는 **로봄 패밀리 표준인 정적 웹 PWA** (러닝봄 구조)로 만든다.

## 이유

- 인수 기준이 "기존 앱들과 같은 형태의 실행 주소" = `https://robom-labs.github.io/calendarbom/`.
  안드로이드 APK 는 이 형태의 주소를 제공할 수 없다.
- 이 작업 환경에는 Android SDK·서명 키·스토어 계정이 없어 네이티브 출시는 사람 확인(새 계약·계정)이
  필요한 영역이다 — 승인 경계상 자동 진행 불가.
- 웹 PWA 는 나머지 세 앱과 같은 배포 파이프라인(GitHub Pages)·패밀리 UI·운영 검증을 그대로 쓴다.

## 대가와 후속

- 웹의 알람은 페이지가 열려 있을 때만 울린다. 이 한계는 앱 안에서 정직하게 안내한다.
- 원 문서의 안드로이드 요구사항(정확한 알람, 부팅 후 재등록, Room 등)은
  `docs/ROADMAP-ANDROID.md` 로 정리해 2차 단계로 남긴다.
- 데이터 모델(제목·날짜·시간·알람 오프셋)은 ICS/JSON 이동을 전제로 설계해
  네이티브 전환 시 마이그레이션 없이 가져갈 수 있게 한다.
