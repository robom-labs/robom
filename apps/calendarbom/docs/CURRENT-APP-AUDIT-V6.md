## V6 관점 문제점 (화면 캡처 기준)

| # | 문제 | 심각도 | V6 해법 |
|---|---|---|---|
| 1 | 시트가 1·2·3 질문을 **한 화면에 전부 펼침** — 320px에서 저장 버튼까지 긴 스크롤 | 높음 | 아코디언: 현재 질문만 펼치고 완료 단계는 한 줄 요약+[변경] |
| 2 | 시간 선택이 **시 12개+분 6개 버튼 상시 노출** — 화면 과밀, 스크롤 유발 | 높음 | 큰 현재 시간 표시 + 빠른 칩(최대 4) + [시간 바꾸기]=±10분/정각/OS 선택기 |
| 3 | 하루 종일 선택 시 **알림이 강제 삭제**됨 (`draft.allDay → reminders=[]`) | 높음 | 하루 종일 일정에도 별도 알림(하루 전 오전 9시 등) 지원 |
| 4 | 제목 preset이 고정 8개 — **최근 사용·지난번과 같게 없음**, 반복 사용자가 매번 같은 입력 | 높음 | 최근 사용(최대 2)+지난번과 같게, 저장 후 최근 템플릿 축적 |
| 5 | **일정 수정 불가** — 완료/삭제만 있음. 시간 하나 바꾸려면 삭제 후 재입력 | 높음 | 일정 상세(수정·복사·다음 일정 만들기·공유) |
| 6 | **반복 없음** — 매일 약, 매년 생일을 지원 못 함 | 높음 | series 반복 엔진(매일/매주/매월/매년/요일) |
| 7 | **사람 개념 없음** — "엄마 병원"을 제목 문자열로만 표현 | 중간 | 지연 등록 사람(이름만), 기본 나, 문맥에서만 질문 |
| 8 | 저장 실수 복구 없음 — 삭제는 confirm()만, **되돌리기 없음** | 중간 | 저장·삭제 후 [되돌리기], 시트 draft 보존 |
| 9 | 알림 탭이 단일 목록 — 오늘/다가오는 구분 없음 | 중간 | 다음 일정/오늘 남은/다가오는/지난 구획 |
| 10 | 공유 없음 — 가족에게 일정 하나를 보낼 방법이 백업 파일뿐 | 중간 | 일정 한 건 텍스트/ICS 공유+미리보기(기본 나만 보기) |
| 11 | 기념일 계산 없음 (100일·1주년 등) | 중간 | 기념일 규칙 엔진 + 선택한 것만 생성 |
| 12 | window.confirm() 사용 — 스타일 불일치·작은 글씨 | 낮음 | 앱 내 확인 시트/되돌리기로 대체 |
| 13 | 시트 스크롤이 길 때 저장 버튼이 화면 밖 | 중간 | 아코디언으로 길이 축소 + 저장 CTA sticky |
