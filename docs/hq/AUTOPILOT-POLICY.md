# v1.2.0 _ 자동운영 등급과 사용량 절약

## 자율 범위 (요청 단위 — intake 6번 질문)

| autonomy | 의미 | 등급 |
|---|---|---|
| research_only | 조사만 하고 보고 | A |
| implement_and_review | 고친 뒤 회장 확인 대기 (기본) | A |
| implement_test_wait_for_deploy | 구현·테스트까지, 배포 대기 | B |
| guarded_auto_deploy | 백업·롤백 준비 후 배포까지 | B |

어떤 값에서도 자동 불가(항상 사람): 유료 계약·광고·결제·새 개인정보 수집·신규 앱 공개·스토어 제출·대규모 브랜드 교체·복구 불가능 삭제·API 과금 전환 (C급 → 결재함).

## 긴급 제어

- `POST /api/control {paused:true}` = 러너가 새 작업을 잡지 않음(화면·트레이 버튼).
- `{intakeClosed:true}` = 새 요청 접수 중단(409).

## AI 사용량 0인 결정론 점검 (§12)

- serve 실행 중 `ROBOM_HQ_WATCH_MINUTES`(기본 10분)마다 build-snapshot 재실행: 운영 URL·버전·SHA·CI·데이터 신선도·이벤트 만료. Codex 호출 없음.
- Codex는 다음일 때만: 승인된 요청 존재 / 회장 수동 실행. (장애 자동진단 트리거는 후속 단계)

## 성장 루틴

매일: 오늘 탭 확인(사람 30초) + 감시기 자동. 매주: 앱 1개 순환 집중 개선을 업무로 등록(회장 또는 총괄팀).
매월: 포트폴리오 우선순위를 결정 기록으로 남김. — 기존 회의실(01~03) 루틴과 병행하며 HQ가 기록·상태의 단일 정본.
