## 앱별 성장 백로그(MED) — ROADMAP 계승, 코덱스 실행 대상

### 야외봄 (robom-labs/outbom)
- [ ] 대형 화면 파일 분리(단일 거대 컴포넌트 → 화면/기능 단위 모듈). 합격: 최대 파일 라인수 하락 + 빌드·테스트 그대로 통과.
- [ ] 날씨·대기질 추천 로직 단위 테스트 추가(골든 케이스). 합격: 추천 함수 입력→출력 고정 테스트 존재.
- [ ] 백그라운드 알림 안정화(권한·재시도·중복 억제). 합격: 알림 스케줄 로직에 재시도·dedup 테스트.

### 청약봄 (robom-labs/homebom)
- [ ] core와 Edge Function의 공고 정규화 로직 **단일 소스화**(중복 제거). 합격: 정규화 코드 1곳, 양쪽이 import.
- [ ] Web Push RFC 상세 설계 반영(방문→알림 허용 흐름). 합격: 구독 등록·해제·전송 경로 테스트.

### 러닝봄 (robom-labs/runningbom)
- [ ] 대회 데이터 갱신 자동화(스케줄 수집 + fingerprint 대조). 합격: 수집 실패/무변경/변경을 구분해 기록.
- [ ] 외부 접수 링크 상태 점검 강화(404·리다이렉트 감지). 합격: 링크 헬스체크가 죽은 링크를 잡는 테스트.

### 공통(6앱)
- [ ] 앱별 **최소 사용 분석** 도구 준비(개인정보 최소 수집, 비공개 endpoint). 합격: 수집 항목·보관 정책 문서 + opt 경로.
- [ ] 방문→알림 허용 **전환율 측정** 지표. 합격: 결정론적 계산 함수 + 테스트.
- [ ] 네 저장소 branch protection·CODEOWNERS 정리(사람 권한 필요 — HUMAN-TASKS와 병행).

## 진행 방식
1. 코덱스 토큰 도착 시: 위 MED 항목을 **한 번에 하나씩** 해당 앱 저장소에서 Loop(목표→기준→수정→원래계약 재검증→종료)로 실행.
2. 각 항목 완료 시 이 파일에서 `[x]`로 체크하고 결과(PR·검증)를 한 줄 남긴다.
3. HQ 본체 개선(robom 저장소)은 코덱스와 무관하게 Claude가 계속 audit→fix→release.
