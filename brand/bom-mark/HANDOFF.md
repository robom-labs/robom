# 로봄 "봄" 로고 마크 — 핸드오프 스펙 (버전 2 확정)

> 이 문서 하나 + 같은 폴더의 `.svg` 파일들만 있으면, 어떤 Claude/개발자든 이 로고 마크를 그대로 구현할 수 있습니다. 다른 세션에 이 파일을 붙여넣고 "이 스펙대로 robom-labs/robom `site/`에 적용해줘"라고 요청하세요.

## 0. 한 줄 요약

로봄 계열사 워드마크의 마지막 **"봄"** 자를, **반듯한 기하 글자(1번 글자체)** + **ㅗ 가로획을 중간 길이로 늘여 그 끝에 벚꽃 개화 + 획을 따라 흩날리는 꽃잎(5번 방식)** 으로 만드는 커스텀 로고 마크. 접두(로/야외/청약/러닝)는 시스템/브랜드 서체 그대로, **"봄"만 이 SVG 마크**로 대체.

## 1. 확정 버전 = "중간 꼬리 · 획 따라 흩날림" (조합 3버전 중 2번)

- 글자체: v5 1번(기하·반듯)
- 꽃/꼬리: v5 5번식 — ㅗ를 늘여 끝에 벚꽃, 꽃잎이 획을 따라 드문드문 떨어짐
- ㅗ 길이: **중간**(너무 길지 않게) = viewBox 좌표에서 x10 → x142

## 2. 브랜드 규칙 (반드시 지킬 것)

1. **정식 이름은 그대로 붙여 씀** — `야외봄`·`청약봄`·`러닝봄`·`로봄`. manifest·검색·스토어·스크린리더용 텍스트는 절대 바꾸지 않는다. 마크는 **로고·아이콘 자산에만**.
2. **마크는 마지막 "봄" 위치에만.** 접두는 서체 그대로.
3. **실루엣 공통 · 색만 앱별.** 하나의 마크 형태를 색만 바꿔 4곳에 쓴다.
4. **로봄 본사 = 알록달록(3색), 각 앱 = 단색.**
   - 로봄: 꽃잎 2:2:1 = 코랄·청록·금 / 흩날림도 3색 순환
   - 야외봄 = 청록 단색 · 청약봄 = 금 단색 · 러닝봄 = 코랄 단색
5. **24px·파비콘**은 봄 글자 없이 **꽃 1송이 단색**으로 축약(`favicon-bloom.svg`).
6. 접근성: `<span class="sr-only">봄</span>`을 마크 옆에 병기(스크린리더가 "봄"을 읽게). SVG는 `aria-hidden` 또는 `aria-label="봄"`.

## 4. 형상 스펙 (viewBox `0 0 200 120`, fill 없음·stroke 기반)

**"봄" 획 (stroke=ink, width=10, linecap/linejoin=round)**
- ㅂ: `M28 8 L28 46` · `M72 8 L72 46` · `M28 28 L72 28` · `M28 46 L72 46`
- ㅗ: 가로 `M10 64 L142 64` · 세로돌기 `M50 64 L50 47`
- ㅁ: `M28 76 L72 76 L72 110 L28 110 Z`

**끝 벚꽃** — 중심 `(148, 60)`, 반지름 18. 꽃잎 5장(72°씩, 살짝 ±2° 비대칭), 중심 `card`원(r2.4)+tone점(r1.2, opacity .5). 꽃잎 path(단위 원점):
`M0 -1.4 C -4.6 -1.8 -6 -6.2 -3.7 -9.2 C -2.5 -10.7 -1.2 -9.4 0 -7.7 C 1.2 -9.4 2.5 -10.7 3.7 -9.2 C 6 -6.2 4.6 -1.8 0 -1.4 Z`

**획 따라 흩날림** — 꽃잎 4장을 `[x,y,회전°,배율]`로 배치:

## 5. 자산 파일 (같은 폴더)

- `bom-robom.svg` — 로봄(알록달록)
- `bom-outbom.svg` — 야외봄(청록)
- `bom-homebom.svg` — 청약봄(금)
- `bom-runningbom.svg` — 러닝봄(코랄)
- `favicon-bloom.svg` — 24px·파비콘용 단색 꽃(코랄)

각 SVG는 hex 색이 박힌 **독립 파일**(라이트 기준). 다크 대응이 필요하면 §6의 컴포넌트(CSS 변수)를 쓰세요.

## 6. robom/site 통합 (Next.js/React 예시)

접두는 텍스트, "봄"은 인라인 SVG. CSS 변수로 테마·앱색 대응:

```tsx
// 로봄 워드마크. 접두는 서체 텍스트, 마지막 "봄"만 커스텀 마크.
function BomWordmark({ prefix, app }: { prefix: string; app: "robom"|"out"|"home"|"run" }) {
  return (
    <span className={`bom-wm ${app}`}>
      {prefix}
      <span className="bom-mark">
        <span className="sr-only">봄</span>
        {/* app==="robom"이면 꽃잎/흩날림을 coral·teal·gold로, 그 외는 var(--tone) 단색으로 */}
        <BomMarkSVG house={app === "robom"} />
      </span>
    </span>
  );
}
```

```css
.bom-wm{ font-weight:900; letter-spacing:-.04em; color:var(--ink,#1d3047); }
.bom-wm .bom-mark{ position:relative; display:inline-block; }
.bom-wm .bom-mark svg{ height:1.18em; vertical-align:-.2em; overflow:visible; }
.bom-wm.out{  --tone:#2f95a0; } /* 야외=청록 */
.bom-wm.home{ --tone:#d99a1f; } /* 청약=금 */
.bom-wm.run{  --tone:#f35f46; } /* 러닝=코랄 */
/* 획은 var(--ink), 단색 앱 꽃잎/흩날림은 var(--tone), 로봄만 3색 하드코딩 */
```

`BomMarkSVG`는 §4 좌표로 그리며, 단색 앱은 꽃잎·흩날림 fill을 `var(--tone)`, 획은 `var(--ink)`; 로봄(house)만 꽃잎을 `#f35f46/#2f95a0/#d99a1f`(2:2:1)로. 다크는 `--ink`/`--tone`을 다크 hex로 재정의.
