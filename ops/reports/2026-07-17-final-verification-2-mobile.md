# 2026-07-17 최종 감사 2 — 모바일·시각·접근성·성능

## 독립 재시작 결과

기능 감사 결과를 전제로 삼지 않고 최신 화면과 반응형 코드를 다시 확인했다.

검사 viewport는 320×568, 360×800, 375×667, 390×844, 412×915, 430×932, 600×960, 768×1024, 1024×768, 1440×1000이다.

## 자격증봄

- 하단 메뉴에 다섯 번째 `시험 달력`이 추가됐고 각 탭은 48px 이상이다.
- 휴대폰에서 달력 아래 agenda, 태블릿에서 두 칸 레이아웃을 사용한다.
- 준비물 행 전체를 누를 수 있고 긴 설명을 자르지 않는다.
- 일반 안내와 공식·교차 검증 준비물의 신뢰 수준을 색만이 아니라 텍스트로 전달한다.
- Chromium·WebKit에서 홈, 달력, 상세, 준비물, 뒤로 복원, offline·PWA update를 재검증했다.

## 캘린더봄

- 날짜를 누를 때 화면 아래 편집기가 자동으로 솟지 않는다.
- `이 날짜에 일정 추가`가 agenda 앞부분과 하단 메뉴 위에서 접근 가능하다.
- 휴대폰 편집기는 전체 화면이고 safe area·VisualViewport를 반영한다.
- 시간 선택은 `시간 미정`, 직접 입력, 최근 시각을 명확히 구분한다.
- 600×960부터 달력과 agenda가 두 칸이며 각 열은 `minmax(0, …)`로 축소 가능하다.
- 200% 확대에서 가로 스크롤과 핵심 문구 잘림이 없다.

## 접근성·성능 판정

- 주요 터치 영역 48px 미만 신규 항목 0건.
- input 16px 미만으로 인한 iOS 자동 확대 신규 항목 0건.
- 가로 스크롤·하단 메뉴 가림·modal 화면 이탈 신규 항목 0건.
- critical/serious axe 위반 신규 항목 0건.
- 새 캘린더·준비물 라이브러리를 추가하지 않아 초기 bundle 의존성 증가를 피했다.
- 실제 필드 Core Web Vitals 수집은 연결되지 않았으므로 LCP·INP·CLS 개선 수치를 만들지 않았다.
