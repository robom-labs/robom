# 로봄 AI 회사 모드

01~03은 동등한 총괄 부서다. 각 방은 전 제품과 운영 영역을 다룰 수 있으며, 번호는 전문 분야나 승인 권한의 차이를 뜻하지 않는다.

## 기본 운용

- 단순한 사실 확인에는 불필요한 팀을 소집하지 않는다. 코드, 디자인, 출시, 수익화, 정책, 여러 저장소에 걸친 비단순 요청은 독립 관점이 있을 때 회사 모드를 적용한다.
- 상위 채팅방 01~03이 이미 병렬이므로, 각 방은 하위 에이전트를 자동으로 대량 생성하지 않는다. 단순 작업은 주 AI가 처리하고, 독립적인 구현·QA·위험검토에 실익이 있을 때만 최소 역할로 나눈다.
- 실제로 참여한 역할만 짧은 팀회의로 사용자에게 공유한다. 내부 사고 과정은 공개하지 않으며, 합의·이견·위험·다음 실행만 요약한다.

## 작업 절차

1. 사용자 목표, 범위, 완료 기준과 안전 예외를 확인한다.
2. 운영 기준 커밋, 상대 AI의 새 회의록, 같은 목표의 PR과 직접 관련된 코드만 증분으로 확인한다.
3. 기존 PR이 있으면 중복 구현하지 않고 검토·보완·QA를 수행한다. 없으면 해당 방의 분리 브랜치와 worktree를 사용한다.
4. 비단순 작업은 구현 범위와 검증 범위를 분리하고 최소 한 역할이 반대 의견과 회귀 위험을 확인한다.
5. 주 AI가 근거와 결과를 직접 확인한 뒤 구현, 테스트, PR, 병합, 기존 배포 파이프라인, 운영 검증을 요청 범위 안에서 연속 수행한다.
6. 중대한 운영 회귀가 확인되면 롤백하고 원인을 정리한다. 고위험 예외만 사용자 판단을 요청한다.

## 검증 원칙

- 직접 관련 테스트와 화면·API 검증을 우선하고, 통합 또는 배포 직전에 필요한 전체 검사를 한 번 실행한다.
- 이미 통과한 같은 커밋의 검사는 관련 코드·설정이 바뀌지 않으면 반복하지 않는다.
- 완료 주장은 코드, 테스트, 화면 또는 GitHub·배포 상태로 직접 확인한다.
